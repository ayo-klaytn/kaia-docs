# カイア・ガバナンス

本書は、カイアの構成要素、運営体制とプロセス、参加基準、報酬の仕組み、入退会手続きなど、カイアの全体的なガバナンスの枠組みを説明するものである。

## 概要

カイア・ガバナンスは3つの主要な構成要素からなる：

- \*\*KAIAコミュニティ:\*\*KAIAトークン保有者を含むコミュニティメンバーは、オンチェーンガバナンスと主要な意思決定プロセスに参加する権利を有する。 また、エコシステム全体に貢献する開発者、サービスプロバイダー、ユーザーも含まれ、ガバナンスだけでなく、エコシステムの継続的発展においても重要な役割を果たす。
- **カイア・ガバナンス・カウンシル(GC):** GCはコミュニティを代表し、直接保有するトークンとコミュニティ・メンバーからの委任された議決権の両方を通じて、ガバナンスの決定に参加します。
- \*\*カイア財団:\*\*カイア財団は、ブロックチェーンとWeb3技術の専門知識を活用してGCの意思決定をサポートし、GCが決定したことを実行する。

カイアGCは、カイア・エコシステムにおける戦略的意思決定の主要機関として機能します。 伝統的な企業、DAO、プロトコル・チーム、コミュニティ・ビルダーなど、多様な主体で構成されている。

GCの組織は、ステークホルダー中心のガバナンスを確保するため、分権化されている。 GC内の議決権は、単一の事業体による独占を防ぐために上限を設け、KAIAの賭け金額と委任金額によって決定される。 この設計により、地域社会の幅広い利益を代表しつつ、透明で効率的な運営が保証される。

\*\*GCの主な責務は以下の通り。

- \*\*新技術の導入、新機能の追加、ブロックチェーン基盤の改善。
- **経済政策：** KAIAの発行と分配、取引手数料の調整、生態系予算の編成。
- \*\*ガバナンスと議決権行使のプロセスの確立と修正、ガバナンス参加者の権利と責任の定義。

ガバナンスの提案は[カイア・ガバナンス・フォーラム](https://govforum.kaia.io/)を通じて提出され、そこで公開討論を経てからオン・チェーン投票に進む。 GCモデルは、すべてのKAIA保有者がカイアのエコシステムに貢献し、影響を与えることができるように設計されています。 エコシステムの進化に伴い、GCはその構造と運営を適応させ続けている。

## オンチェーン・ガバナンス

オンチェーンガバナンスは、スマートコントラクトを通じてステークホルダー間の意思決定を自動化し、透過的に実行する。 このアプローチにはいくつかの利点がある：

- **透明性：** すべての統治活動はブロックチェーン上に記録され、誰でも検証可能である。
- **誠実さ：** 投票はスマート・コントラクトによって処理されるため、改ざんができない。
- **否認防止:** 署名された票はチェーン上に永久に記録され、否認されることはありません。
- \*\*結果は自動的に実行され、決定が無視されることはない。

カイア・ブロックチェーンは、このような特性を持つオンチェーン・ガバナンス・システムを実装している。 投票権は賭けられたKAIAの量に比例し、過度の集中を防ぐために上限が設定されている。 KAIA保持者は、議決権をGC会員に委任することもできる。

### 投票権の計算

- GCの[議決権](https://square.kaia.io/Proposal?tab=LEADERBOARD)はステークされたKAIAの量に比例する(5M KAIAにつき1票)。
- 公開委任の仕組みにより、GC会員でないKAIA保有者は、その議決権をGC会員に委任することができる。
  - 委任されたトークンはGC自身の保有トークンと合算され、議決権は同じ基準（500万KAIAにつき1票）で計算される。
- 単一の企業が議決権を独占することを防ぐため、上限が設定されている。
  - \*\*最大得票数：\*\*\*有効なGC会員総数-1（例：有効な会員が40名の場合、上限は39票）。

### カイア・ガバナンス・プロセス

1. \*\*議論：\*\*誰でもガバナンス・フォーラム（<https://govforum.kaia.io>）を通じて提案を提出することができる。
2. **提案の登録と投票：** 提案がガバナンス・フォーラムでGCメンバーから少なくとも1件の肯定的なフィードバックを受けると、その提案は登録され、投票のために開放されます。
3. \*\*承認されたプロポーザルは実行される。

![](/img/misc/gov-process.jpg)

オンチェーンに登録された提案は、投票が終了するまでいくつかのステータスを経て進行する：

- \*\*保留：登録後、投票開始を待つ。
- \*\*現在投票中です。
- **可決：** 定足数と多数決により承認された。
- \*\*失敗:\*\*十分な承認が得られなかった。
- \*\*Queued:\*\*承認されたが、実行は保留されている。
- \*\*プロポーザルは完全に執行された。

![](/img/misc/gov-process-2.png)

標準的な議決権行使期間は7日間だが、議案によっては追加の手続きが必要となる場合もある。  
例えば、**テキストのみのプロポーザル**（[例](https://square.kaia.io/Proposal/Detail?id=31)）は、投票終了後に結論を出す。 しかし、**パラメーター変更プロポーザル**（[例](https://square.kaia.io/Proposal/Detail?id=26)）は、承認だけでなく、2～16日遅れて関連取引を実行する必要がある。

パラメータ変更の詳細については、[KIP-81：オンチェーン・ガバナンス投票の導入](https://kips.kaia.io/KIPs/kip-81)を参照のこと。 緊急の場合は、このスケジュールを短縮することができる。

### 定足数と承認基準

- **最低定足数：** Count Quorum（GC会員の少なくとも3分の1が参加）またはPower Quorum（総投票権の少なくとも3分の1が投票）のいずれか。
- **承認：**\* 参加票の過半数の承認が必要。
  - **棄権：** 投票参加としてカウントされるが、賛成票と反対票の計算からは除外される。

### 欠席ペナルティ

欠席ペナルティ・ポリシーは、「KGP-31：GC投票参加を奨励するガバナンス・ポリシー」(https://govforum.kaia.io/t/kgp-31-governance-policy-to-encourage-gc-voting-participation/808)で紹介され、承認された。 連続3回\*\*の投票に参加できなかった会員は、**7日間**ブロック報酬から除外されます。

## GCの資格、責任、報酬

### 参加資格

- 最低**5,000,000 KAIA賭け**。
- Kaiaノード\*\*を操作する（[システム要件](https://docs.kaia.io/nodes/core-cell/system-requirements/)を参照）。
- ガバナンスへの積極的参加
  - 討論と投票への参加
  - 定例会議および特別会議への出席

### 責任

#### ネットワークのセキュリティと安定性のためのノード運用

- ノード運用  
  高可用性と最小限のダウンタイムでネットワークのコンセンサスをサポートするため、継続的で安全なノード運用を維持する。
- モニタリングとアラート  
  ネットワークの脅威や障害を監視し、エコシステムの安定性を確保するために迅速に対応します。

#### ガバナンスへの参加とKEFの監督

- GCメンバーは、GCまたはより広範なコミュニティからの提案を検討し、決定する。 GCの議題には、技術的な詳細や実施方法は含まれていないが、方向性、方針決定、トークノミクスは含まれている。 GCは、提案の提出、議論、投票結果の実施など、オンチェーンガバナンスの投票プロセスに積極的に参加しなければならない。
  - GC会合は**毎月第3水曜日**に開催され、議題がない場合は省略されることもある。
  - 緊急の案件については、臨時総会を招集することができる。
- **カイア・エコシステム・ファンド（KEF）**は、インフラ強化、開発者支援、間接投資を通じて生態系の持続可能性を支援する。 GCは**KEF予算**の承認と**資金執行の監督**に責任を負う。

#### コミュニティ参加と透明性

- 多様な利害関係者の利益を代表し、誠実さを守り、利益相反を防止し、カイアのエコシステムの最善の利益のために行動する。
- 透明でタイムリーなコミュニケーションチャネルを維持し、会議の要約と決定事項を明確に開示する。
- 代表団を通じてKAIAトークン保有者の積極的な参加を促す。

### 報酬

現在の報酬体系は[KIP-82](https://kips.kaia.io/KIPs/kip-82)で提案され、承認された。 これは、**提案者報酬**と**ステーキング報酬**で構成されている。

- **提案者の報酬:**  
  すべてのGCメンバーは少なくとも5M KAIAをステークし、CN/PNノードを運用し、ブロック生産に等しく参加する。 各GCメンバーは、提案された1ブロックにつき**0.96KAIAを受け取る**。
- **賭け金報酬:**  
  KAIAを5M以上賭けているGC会員は、超過分の賭け金に比例した報酬を受け取ります。 ブロックあたり**3.84KAIAの分配**から、各メンバーの超過賭けKAIAのシェアに基づいて報酬が配分されます。

## オンボーディング・プロセス

オンボーディング・プロセスは次のような段階を踏む：

1. **Expression of Interest:**  
   候補者は、[ガバナンス・フォーラム](https://govforum.kaia.io/)およびカイア・コミュニケーショ ン・プラットフォームの[招待状請求](https://docs.google.com/forms/d/e/1FAIpQLSdj-N63AAAWhzPh8GIJQGOUrKk_ppWtzDwUyflq532IdlQ3Jw/viewform)を通じて、GCへの参加に関心を表明する。
2. **プレ・オンボーディング・サーベイとツール配布:**  
   候補者はプレ・オンボーディング[サーベイ](https://docs.google.com/forms/d/1o1HUe3SNLRI_txqymSICGoP48Lq71HEz0PxIZZBynNo/preview)を完了し、ガバナンス活動の経験と運用能力を評価する。 財団はノード管理と公開委任のために**Klaytool**と関連ガイドを提供している。
3. **Private network/Testnet/Mainnet Onboarding:**  
   オンボーディングは、メインネットに進む前にプライベートネットワークとテストネット環境から始まる。 このプロセスには通常**3週間**ほどかかりますが、詳細なスケジュールは必要に応じて調整されます。
4. **最終確認と発表:**  
   終了後、候補者は正式にGC会員となります。 オンボーディングの状況は[カイア・スクエア](https://square.kaia.io/GC?tab=HISTORY)で確認でき、ガバナンス・チャンネルを通じてメンバーシップが発表され、投票権が有効になる。

## オフボーディング・プロセス

オフボーディング・プロセスは、ネットワークの安定性、完全性、コミュニティに対する透明性を保証する。 これは、会員が**自発的に退会**した場合、**不正行為**により解雇された場合、**参加要件を満たさない場合**に適用される。 ノードが適切に動作していない場合、ペナルティや**強制的なオフボーディング**につながる可能性がある。

\*\*オフボーディングの手順

1. **退会または解任の通知：**  
   GC会員が財団に退会の通知をした場合、財団はそれを[ガバナンス・フォーラム](https://govforum.kaia.io/)に掲載し、正式な退会手続きを開始します。
2. **カイアスクエア経由の処理：**  
   【カイアスクエア】(http://square.kaia.io)の管理者が退会日を指定し、GC枠を解除する。
3. **バリデーター、アドレス帳、BLS除去：**\*。

- Foundation ノードの演算子は、そのメンバを Validator リストから削除します。
- 財務チームは、トークン・マネージャー（TM）を介してアドレス帳から GC の記録を削除します。
- コアチームはGCのBLSキーを取り外す。

4. **ノードのシャットダウンと公示:**  
   上記の手順の後、メンバーのノードは安全にシャットダウンすることができます。 オフボーディングは、ガバナンス・フォーラムや公式のコミュニケーション・チャンネルを通じて公に発表される。
